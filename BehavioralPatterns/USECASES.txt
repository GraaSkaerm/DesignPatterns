BEHAVIORAL PATTERNS

1::CHAIN PATTERN

	USE CASES::
		Brug til at tjekke eller håndter data med en bestemt rækkfølge.
		USE THE PATTERN WHEN IT'S NEEDED TO EXECUTE SEVERAL HANDLERS IN A PARICULAR ORDER

2::COMMAND PATTERN
	USE CASES::
		Brug til input events.
		USE TO ENCAPSULATE A REFERENCE TO AN ACTION/COMMAND IN AN OBJECT SO IT DOSNT NEED THE LOGIK ITSELF

3::ITERATOR PATTERN
	USE CASES::
		Brug til at gennemgå en samling med en bestemt tilgang.
		USE TO DEFINE HOW TO GO THROUGH A COLLECTION


4::MEDIATOR PATTERN
	USE CASES::
		Brug til at opdele objeker med mange referencer.
		USE WHEN REFERENCES HINDRES THE REUSEABILITY OF AN OBJECT
		USE WHEN THE NUMBER OF OBJECTS GROWS SO LARGE THAT CONTAINING REFERENCES TO THEM IS DIFFICULT


5::MOMENTO PATTERN
	USE CASES::
		Brug til at gemme data.
		USE WHEN AN APP SHOULD AUTO SAVE

	NOTES::
		ORIGINATOR - THE OBJECT WE WANT TO SAVE
		MOMENTO - HOLDS THE STUFF THE ORIGINATOR/OBJECT SHOULD SAVE
		CARETAKER - HOLDS ALL SAVES AND MAKES IT POSSIBLE TO LOAD A SAVE


6::OBSERVER PATTERN
	USE CASES::
		USE WHEN AN EVENT SHOULD NOTIFY LISTENERS


7::STATE PATTERN
	USE CASES::
		USE WHEN AN OBJECTS HAS A LOT OF STATES


8::STRATEGY PATTERN
	USE CASES::
		USE WHEN YOU WANT DIFFERENT VARIANTS OF AN ALGORITHM TYPE
		USE WHEN A LOT OF SIMILAIR CLASSES ONLY DIFFER IN THE WAY THE CODE IS EXECUTED
